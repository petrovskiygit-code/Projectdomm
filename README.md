# Тестирование модулей банковского приложения

## Обзор тестирования

Этот проект содержит комплексные тесты для модулей:
- **widget** - функции форматирования и маскирования
- **processing** - функции обработки данных

## Покрытие тестами

### Модуль widget

#### Функция `get_mask_card_number`
- Стандартные номера карт (16 цифр)
- Короткие/длинные номера (граничные случаи)
- Пустые строки и None
- Строки с пробелами и специальными символами
- Номера AMEX (15 цифр)

#### Функция `get_mask_account`
- Стандартные номера счетов (20 цифр)
- Короткие/длинные номера
- Пустые строки
- Строки с пробелами

#### Функция `mask_account_card`
- Автоматическое распознавание типа (карта/счет)
- Параметризованные тесты разных платежных систем
- Некорректные входные данные
- Пустые строки и None

#### Функция `get_date`
- Различные форматы дат
- Граничные случаи (пустые строки)
- Нестандартные строки с датами
- Обработка ошибок парсинга

### Модуль processing

#### Функция `filter_by_state`
- Фильтрация по различным статусам
- Отсутствие элементов с искомым статусом
- Пустые списки и None
- Чувствительность к регистру

#### Функция `sort_by_date`
- Сортировка по убыванию/возрастанию
- Одинаковые даты (стабильность сортировки)
- Невалидные форматы дат
- Отсутствующее поле даты
